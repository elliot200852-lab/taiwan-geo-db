# 「認識臺灣」臺灣地理資料庫 — 研究與規劃報告

> 2026-07-14 · 三路平行研究（政府開放資料 API／地圖圖資與技術／圖片素材授權）＋ TCMB 本機索引盤點的彙整。
> 目標：配合 5A 主課程「認識臺灣」，做一個互動地圖網站——點擊縣市／鄉鎮顯示該區人文與自然地理內容與圖片。

---

## 一、結論先講

**素材面完全可行，且幾乎全部免費、授權乾淨（OGDL／CC，只需顯名標示）。**
最大的工程量不在「找資料」而在「合資料」：政府沒有一份把人口＋農產＋景點＋地形＋氣候按鄉鎮合併好的一站式資料集，需要自建 build 期 ETL，以**鄉鎮市區代碼**為主鍵把多來源 join 成我們自己的資料庫。這正是本專案的核心價值——做出全臺第一份「教育視角的鄉鎮地理卡片庫」。

技術面推薦：**靜態站（Leaflet + TopoJSON，無後端）**，與 CreatorHub link-out app 模式完全相容。

## 三、技術方案推薦

**Leaflet + TopoJSON，純靜態站，無後端。**

- 點擊互動內建、觸控原生佳、學習曲線低、~150KB gz
- 不掛 raster 底圖 → 整站離線自包含，Firebase Hosting 直接放
- 體積策略：build 期 mapshaper 簡化＋quantize；**縣市層先載（<100KB），點擊縣市後懶載該縣的鄉鎮層**
- 資料架構：build 期 ETL 產出 `data/{縣市代碼}.json`（含該縣各鄉鎮卡片），前端按需 fetch——368 鄉鎮不會一次塞爆
- 備選：若之後想要資料著色圖（choropleth），ECharts 最省事；D3 只在要精緻客製視覺時才值得

**與 CreatorHub 相容性**：完全符合既有 link-out app 模式（`public/apps/<slug>/`＋channels.json 加 appUrl＋封面圖），不需要動 CreatorHub 架構。

### 參考案例
- 微笑台灣 319 鄉（天下）— 內容架構最佳參照
- PlotDB 縣市→鄉鎮下鑽範本
- 教地理 twgeoedu — 教育場景直接對標

## 四、分期建議

| 期 | 內容 | 產出 |
|---|---|---|
| **P0 資料骨幹** | ETL：人口#8410＋農情#7302＋景點#7777 CSV 按鄉鎮代碼 join；taiwan-atlas 圖資簡化 | 368 鄉鎮基本卡（數據欄位齊） |
| **P1 縣市版上線** | Leaflet 地圖＋22 縣市頁（人文／自然摘要＋精選圖片）；圖片管線（TCMB＋Commons，含自動出處標示） | 可給 5A 用的第一版 |
| **P2 鄉鎮下鑽** | 點縣市→鄉鎮層懶載；鄉鎮卡片逐縣充實（可按主課程進度，教到哪個區域做哪區） | 逐步長成完整資料庫 |
| **P3 深化圖層** | DTM 地形統計（各鄉平均海拔／最高峰）、水文流域圖、生態 TBIA、氣候 | 自然地理進階內容 |

「地方特色敘事」（這個鄉鎮以什麼聞名、族群、傳說）沒有現成 API——建議用 AI 管線草擬＋David 審稿的模式（同 hs-book-edu-summary 的做法），這是內容品質的關鍵人工環節。

## 五、風險與注意

1. **授權顯名**：所有政府資料與 NLSC 圖資都要求頁尾標示來源機關；圖片一律程式化存 license＋author 自動生成出處。
2. **圖資現行性**：避開 g0v 舊圖資；用 taiwan-atlas 或內政部 2025-12 版。
3. **體積**：鄉鎮層未簡化可達數十 MB，必須簡化＋分縣懶載。
4. **待人工確認**：TDX V2 授權標示字樣、觀光多媒體批次下載方式、開放博物館檢索 API 端點。
